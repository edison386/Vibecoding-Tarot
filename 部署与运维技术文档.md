- 公网 IP：`42.121.252.175`
- 域名：`lovetarot.fun`

## 7. 当前线上配置

### 7.1 `tarot-app.service` 推荐结构

当前线上服务建议保持如下结构：

```ini
[Unit]
Description=Tarot App
After=network.target

[Service]
Type=simple
WorkingDirectory=/opt/tarot-app
Environment=NODE_ENV=production
Environment=PORT=3000
Environment=HOST=127.0.0.1
Environment=SPARK_API_PASSWORD=你的新密钥
Environment=SPARK_MODEL=generalv3.5
Environment=AI_READING_TIMEOUT_MS=15000
ExecStart=/usr/local/bin/node /opt/tarot-app/src/server.js
Restart=always
RestartSec=3
User=root

[Install]
WantedBy=multi-user.target
```

如果使用 Ultra 模型，建议改为：

```ini
Environment=SPARK_MODEL=4.0Ultra
Environment=AI_READING_TIMEOUT_MS=45000
```

### 7.2 当前 Caddy 配置建议

```caddyfile
www.lovetarot.fun {
    redir https://lovetarot.fun{uri} permanent
}

lovetarot.fun {
    reverse_proxy 127.0.0.1:3000
}
```

说明：

- 第一段用于 `www` 自动跳转到主域名
- 第二段用于正式域名站点
- 如遇域名异常，可临时加回公网 IP 站点块作为应急入口

## 8. 首次部署流程

### 8.1 上传代码

将项目上传到服务器后，统一放在：

```bash
/opt/tarot-app
```

### 8.2 安装 Node

使用官方二进制包安装：

```bash
cd /tmp
curl -fsSLO https://nodejs.org/dist/v20.20.1/node-v20.20.1-linux-x64.tar.xz
sudo tar -xJf node-v20.20.1-linux-x64.tar.xz -C /usr/local --strip-components=1
node -v
npm -v
```

### 8.3 启动 systemd 服务

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now tarot-app
sudo systemctl status tarot-app --no-pager
```

### 8.4 验证本地服务

```bash
curl http://127.0.0.1:3000/healthz
```

正常返回：

```json
{"ok":true}
```

### 8.5 安装并配置 Caddy

写入 `Caddyfile` 后执行：

```bash
sudo systemctl reload caddy
sudo systemctl status caddy --no-pager
```

### 8.6 验证反向代理

```bash
curl -I http://lovetarot.fun
curl -I https://lovetarot.fun
```

## 9. 日常发布流程

每次代码更新后，建议按以下流程发布。

### 9.1 备份当前版本

如果是重要版本，先备份：

```bash
sudo cp -a /opt/tarot-app /opt/tarot-app.bak.$(date +%F-%H%M%S)
```

### 9.2 上传新代码

将本地最新代码上传到服务器临时目录，例如：

```bash
/home/deploy/TAROT
```

然后覆盖正式目录：

```bash
sudo rsync -av --delete /home/deploy/TAROT/ /opt/tarot-app/
```

如果服务器没有 `rsync`，可先安装：

```bash
sudo apt update
sudo apt install -y rsync
```

### 9.3 重启服务

```bash
sudo systemctl restart tarot-app
sudo systemctl status tarot-app --no-pager
```

### 9.4 验证服务

```bash
curl http://127.0.0.1:3000/healthz
curl -I https://lovetarot.fun
```

### 9.5 查看日志确认无异常

```bash
sudo journalctl -u tarot-app -n 100 --no-pager
```

### 11.5 域名异常说明

如果域名处于：

- `serverHold`

则通常表现为：

- 无法正常解析
- `dig` 没有结果
- `curl` 报 `Could not resolve host`

此时不是服务器配置错误，而是域名注册状态未恢复。

## 14. 常见问题与处理方式

### 14.1 `Caddy works!`

原因：

- Caddy 还在使用默认欢迎页配置

处理：

- 检查 `/etc/caddy/Caddyfile`
- 确保已经写入业务配置并执行 `sudo systemctl reload caddy`

### 14.2 `curl: (6) Could not resolve host`

原因：

- 域名 DNS 未生效
- 域名处于 `serverHold`

处理：

- 先查 `dig`
- 再确认域名注册和实名状态

### 14.3 `analysis_source: "template"`

原因：

- 星火接口访问失败
- AI 密钥错误
- 超时或模型配置异常

处理：

- 检查 `tarot-app.service` 中环境变量
- 检查日志：

```bash
sudo journalctl -u tarot-app -n 100 --no-pager
```

### 14.4 本地健康检查正常，但公网打不开

检查顺序：

1. Caddy 是否正常运行
2. `Caddyfile` 是否正确
3. 阿里云安全组是否放行 `80/443`
4. `ufw` 是否放行 `80/443`
5. 域名 DNS 是否正确
6. 域名是否处于 `serverHold`

## 15. 安全注意事项

### 15.1 AI 密钥

`SPARK_API_PASSWORD` 属于敏感信息。

要求：

- 不要写进前端代码
- 不要提交到 Git 仓库
- 不要出现在公开截图中

如果密钥曾经暴露，必须：

- 立即在讯飞后台作废旧密钥
- 替换为新密钥
- 重启 `tarot-app`

### 15.2 SSH 账号

建议长期使用普通运维账号，不建议依赖云平台辅助账号做常规运维。

### 15.3 权限收敛

如果后续进入正式运营阶段，建议进一步优化：

- 使用专门应用用户运行 Node 服务，而不是 `root`
- 收紧 SSH 来源 IP
- 定期更新系统安全补丁
